信实文档处理工作台 用户使用指南

版本说明
本指南基于 2026-04-01《项目结构分析.md》以及当前系统实际页面、接口和任务流程整理。
本文面向日常业务使用，内容以“当前系统实际可用功能”为准，不沿用旧版本说明。

一、平台简介
信实文档处理工作台是一个基于浏览器的在线文档处理平台，无需安装本地客户端。用户在浏览器中上传文件、设置参数、提交任务后，系统会把任务统一写入任务队列，在后台依次执行，并在任务工作台中提供进度查看、日志查看、结果下载和取消操作。

当前平台主要包含以下模块：
1. 证件翻译工作台
2. 数字专检
3. 中翻译专检
4. 多语对照记忆
5. 不可编辑文档预处理

其中，证件翻译工作台又整合了以下四类证件处理入口：
1. 身份证 OCR 翻译
2. 驾驶证翻译
3. 营业执照翻译
4. 通用证件翻译（不含人像）

二、访问入口
使用时，请将“部署地址”替换为当前实际服务器地址，例如本地、局域网地址或云端域名。

主要入口如下：
1. 首页：部署地址/
2. 任务工作台：部署地址/dashboard
3. 证件翻译工作台：部署地址/certificate-translation
4. 身份证 OCR 翻译：部署地址/ocr
5. 驾驶证翻译：部署地址/drivers-license
6. 通用证件翻译：部署地址/doc-translate
7. 数字专检：部署地址/number-check
8. 中翻译专检：部署地址/zhongfanyi
9. 多语对照记忆：部署地址/alignment
10. 不可编辑文档预处理：部署地址/pdf2docx

说明：
1. 证件类业务建议优先从“证件翻译工作台”进入，再切换到对应页签。
2. 任务统一在“任务工作台”查看，不需要每次回到原页面等结果。

三、通用使用流程
1. 打开对应功能页。
2. 上传文件。
3. 按页面要求选择模式、语言、模型或其他参数。
4. 点击“开始处理”或“开始翻译”。
5. 任务提交成功后，系统会进入统一任务队列。
6. 如需持续查看进度，可停留在当前页面；如需统一查看全部任务，可进入“任务工作台”。
7. 任务完成后，在当前结果页或任务工作台中下载输出文件。

通用说明：
1. 任务提交后，即使关闭当前页面，后台任务仍会继续执行。
2. 如果状态显示“排队中”，说明前面有其他任务正在处理，属于正常现象。
3. 对“排队中”或“处理中”的任务，可以在任务工作台发起取消。取消不是立刻中断，而是会在当前步骤结束后停止。

四、证件翻译工作台
证件翻译工作台是当前证件类业务的统一入口，适合先按证件类型分流，再进入具体处理页。

4.1 身份证 OCR 翻译
适用场景：
用于身份证正反面图片识别与翻译。

支持格式：
JPG、JPEG、PNG、BMP、TIF、TIFF 等常见图片格式。
页面支持多选上传，单文件大小建议不超过 50MB。

主要参数：
1. 源语言
2. 目标语言
3. 是否生成 OCR 可视化图
4. 多图时可为每张图片选择“自动判断 / 正面 / 背面”

输出结果：
1. 翻译后的图片
2. OCR 原始数据
3. 翻译数据
4. OCR 可视化图（勾选后生成）

使用说明：
1. 上传身份证图片。
2. 选择源语言和目标语言。
3. 如需核对识别区域，可勾选生成 OCR 可视化图。
4. 点击开始处理。
5. 完成后下载翻译图片或相关数据文件。

特别说明：
当前该页面按身份证场景接入，请不要再按旧版说明将它作为通用证件入口使用。
其他无固定模板证件，请优先使用“通用证件翻译”或对应专项页面。

4.2 驾驶证翻译
适用场景：
用于驾驶证主页、副页、附页等图片的识别与标准化译文生成。

支持格式：
PNG、JPG、JPEG、BMP、TIF、TIFF、WEBP。
单次请求最多支持 20 张图片。

处理模式：
1. 单图处理：1 张图片输出 1 份译文
2. 多图合并：多张同一证件图片合并输出 1 份 Word
3. 多图批量：多张图片分别输出多份 Word

输出结果：
DOCX 译文文件。

使用说明：
1. 上传驾驶证图片。
2. 选择处理模式。
3. 点击开始处理。
4. 处理完成后下载 DOCX 结果。

使用建议：
1. 同一份驾驶证的多张图片，请使用“多图合并”。
2. 不同驾驶证的多张图片，请使用“多图批量”。

4.3 营业执照翻译
适用场景：
用于营业执照图片识别、字段抽取、公司名称确认和统一格式英文译文生成。

支持格式：
PNG、JPG、JPEG、BMP、TIF、TIFF、WEBP、GIF。

主要参数：
1. 识别模型

特殊步骤：
系统会先识别公司中文名称，并给出建议英文名称。用户确认后，系统才会正式提交营业执照翻译任务。

输出结果：
营业执照译文 DOCX。

使用说明：
1. 上传营业执照图片。
2. 选择识别模型。
3. 点击开始处理。
4. 根据页面提示确认公司英文名称，必要时可手工修改。
5. 提交正式任务后等待系统处理。
6. 完成后下载营业执照译文 DOCX。

使用建议：
请上传完整、清晰、无遮挡的原图，尽量不要裁掉二维码、印章和边缘字段。

4.4 通用证件翻译（不含人像）
适用场景：
适合没有固定模板、主要为文字信息的通用证件或材料。
该模块支持一份文件同时翻译成多种语言。

支持格式：
PDF、JPG、JPEG、PNG、BMP、GIF、WEBP、TIF、TIFF。
批量模式最多支持 50 个文件。

主要参数：
1. OCR 模型
2. 源语言
3. 目标语言，可多选

输出结果：
1. OCR 原始文本 TXT
2. 每个目标语言对应的 DOCX 译文
3. 每个目标语言对应的译文 TXT

使用说明：
1. 上传 PDF 或图片文件。
2. 选择 OCR 模型。
3. 选择源语言。
4. 勾选一个或多个目标语言。
5. 点击开始处理。
6. 完成后按语言分别下载 Word 和译文文本。

使用建议：
该模块更适合不含人像、没有固定排版模板、但需要快速导出多语种 Word 的材料。

五、专项处理模块
5.1 数字专检
适用场景：
用于原文与译文中的数字、编号、页眉页脚等数值一致性检查。

模式说明：
1. 双文件模式：上传原文 DOCX 和译文 DOCX
2. 单文件模式：上传一个双语对照文件

支持格式：
1. 双文件模式：仅支持 DOCX
2. 单文件模式：支持 DOCX、PDF、XLSX、PPTX

输出结果：
1. DOCX 输入时可输出修订版文档
2. 正文报告 JSON
3. 页眉报告 JSON
4. 页脚报告 JSON

使用说明：
1. 选择检查模式。
2. 按模式上传文件。
3. 点击开始处理。
4. 处理完成后下载修订版和报告文件。

使用建议：
1. 如果原文和译文分别保存，请使用双文件模式。
2. 如果文件本身是双语对照版，请优先使用单文件模式。

5.2 中翻译专检
适用场景：
用于原文与译文的深度比对、规则审查、修复输出和报告导出。

模式说明：
1. 双文件模式：上传原文和译文两个文件
2. 单文件模式：上传一个双语对照文件

支持格式：
1. 双文件模式：DOCX、DOC、PDF、XLSX、XLS、PPTX
2. 单文件模式：DOCX、DOC、PDF、XLSX、XLS、PPTX

可选功能：
1. 可上传规则文件
2. 勾选“使用 AI 生成规则”时，必须上传规则文件，或在规则编辑器中填写规则内容

规则编辑说明：
页面中的“编辑规则文件”支持读取默认规则或自定义规则。
保存后的内容仅对本次会话任务生效，不会写回服务器磁盘文件。

输出结果：
根据输入文件类型不同，系统会输出以下一种或多种结果：
1. 修复后文档
2. 批注版 PDF
3. 批注版 Excel
4. 批注版 PPTX
5. 正文、页眉、页脚的 JSON 报告
6. 正文、页眉、页脚的 Excel 报告

使用说明：
1. 选择模式并上传文件。
2. 如需使用规则文件，按需上传。
3. 如需 AI 规则，勾选对应选项。
4. 点击开始处理。
5. 完成后下载修复结果和报告文件。

5.3 多语对照记忆
适用场景：
用于把原文与译文自动对齐成 Excel 对照表，方便术语整理、记忆库沉淀和项目复盘。

支持格式：
DOCX、DOC、PPTX、XLSX、XLS。

主要参数：
1. 原文语言
2. 译文语言
3. 模型
4. 是否启用后处理细粒度分句

高级参数：
页面提供分割阈值和缓冲字符设置，仅对 DOCX 生效。
如果没有特殊需求，建议保留默认值。

输出结果：
1. 对齐结果 Excel
2. 部分任务还会提供中间处理文件下载

使用说明：
1. 上传原文文件和译文文件。
2. 选择源语言和目标语言。
3. 选择模型。
4. 一般情况下保持“启用后处理细粒度分句”为勾选状态。
5. 点击开始处理。
6. 完成后下载 Excel 对照结果。

使用建议：
当源语言为英文等西方语言时，建议保留细粒度分句选项，以获得更细的对齐结果。

5.4 不可编辑文档预处理
适用场景：
用于把扫描件、图片版 PDF、票据、表格型页面等不可编辑材料整理成可编辑 Word。

支持格式：
PDF、PNG、JPG、JPEG、BMP、GIF、WEBP。
批量模式最多支持 50 个文件。

重要限制：
单个 PDF 最多支持 100 页，超过限制将无法提交。

主要参数：
1. 转换模型

输出结果：
1. DOCX 文件
2. 原始 OCR 文本 TXT

使用说明：
1. 上传 PDF 或图片。
2. 选择转换模型。
3. 点击开始处理。
4. 处理完成后下载 Word 和 OCR 文本。

使用建议：
1. 普通材料可先尝试快速版模型。
2. 复杂表格、版式混乱或识别要求较高时，可尝试增强版模型。

六、任务工作台使用说明
任务工作台入口为：部署地址/dashboard

主要功能如下：
1. 查看任务统计
顶部可查看全部任务、排队中、处理中、已完成、失败、已取消的数量。
点击统计卡片可按状态筛选任务。

2. 任务筛选
支持按任务类型筛选，也支持按文件名关键词搜索。

3. 查看任务详情
点击任务行或“详情”按钮，可查看：
1. 任务编号
2. 任务类型
3. 提交时间、开始时间、完成时间
4. 当前进度和处理提示
5. 预计完成时间
6. 输入文件
7. 输出文件
8. 错误信息
9. 运行日志

4. 下载结果
已完成任务可在任务详情中直接下载输出文件。

5. 取消任务
对排队中或处理中任务，可在列表页或详情页发起取消。
取消请求提交后，系统会在当前步骤结束后停止。

6. 页面关闭说明
功能页关闭后，任务不会丢失，仍可在任务工作台继续查看和下载。

七、推荐操作习惯
1. 上传前先检查文件是否清晰、完整，尤其是印章、二维码、页眉页脚、表格边框和证件边缘。
2. 批量处理前，建议先用 1 份样例文件验证参数和输出效果。
3. 对需要正式交付 Word 的场景，下载后建议人工复核一次排版，重点检查表格、分页、字号和对齐。
4. 遇到复杂版面或识别不稳定时，优先尝试更强模型，而不是反复使用同一参数重试。

八、常见问题
1. 任务一直显示“排队中”
说明前面还有其他任务在执行，属于正常现象，请稍后刷新任务工作台查看。

2. 上传后提示格式不支持
请检查文件扩展名是否在当前模块支持范围内。不同模块支持的格式不同，不能混用。

3. PDF 无法提交到“不可编辑文档预处理”
请先检查 PDF 页数是否超过 100 页。超过限制的 PDF 无法在该模块提交。

4. 点了取消，为什么任务没有立刻停止
取消属于异步生效，系统会在当前步骤结束后停止。

5. 中翻译专检里编辑过的规则，为什么下次打开没有保留
规则编辑保存的是“本次任务会话规则”，不会写回服务器磁盘文件。

九、当前版本特别说明
1. 证件类业务现在优先从“证件翻译工作台”进入，再按页签切换到身份证、驾驶证、营业执照或通用证件。
2. “身份证 OCR 翻译”当前按身份证场景接入，不再按旧版说明作为通用证件入口使用。
3. 所有任务最终都会进入统一任务队列，结果也都可以在任务工作台统一追踪和下载。
